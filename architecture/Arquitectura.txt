MARA — Monitor y Asesor de Finanzas Personales y Ahorro
Arquitectura del proyecto (estado actual)

finexpert/
│
├── backend/
│   ├── app.py                          ← Punto de entrada Flask: /diagnose, /explain, /health
│   ├── config.py                       ← Variables de entorno con valores por defecto
│   │
│   ├── engine/
│   │   ├── inference_engine.py         ← Ciclo Match-Select-Execute (RETE simplificado)
│   │   ├── working_memory.py           ← Clase WorkingMemory (hechos de sesión activa)
│   │   ├── agenda.py                   ← Evaluación de condiciones y conjunto de conflicto
│   │   ├── conflict_resolver.py        ← Estrategia: prioridad → especificidad → certeza
│   │   └── explainer.py                ← Trazabilidad: bitácora → lenguaje natural
│   │
│   ├── knowledge/
│   │   ├── knowledge_base.py           ← Cargador y validador de reglas JSON
│   │   ├── fact_base.py                ← Umbrales, rangos y constantes del dominio
│   │   ├── rules/
│   │   │   ├── diagnostico.json        ← 20 reglas (D01-D20): situacion, alertas secundarias
│   │   │   ├── deuda.json              ← 15 reglas (DP01-DP15): DAI, estrategia de pago
│   │   │   ├── ahorro.json             ← 15 reglas (AH01-AH15): fondo emergencia, nivel ahorro
│   │   │   ├── perfil_riesgo.json      ← 10 reglas (PR01-PR10): edad, horizonte, tolerancia
│   │   │   ├── inversion.json          ← 10 reglas (IN01-IN10): instrumento, diversificación
│   │   │   └── pronostico.json         ← 10 reglas (FP01-FP10): tendencias, proyección 6m
│   │   └── ontology/
│   │       ├── conceptos.json          ← Jerarquía de conceptos financieros del dominio
│   │       └── umbrales_financieros.json ← Constantes CONDUSEF/CNBV/Banxico/ENIF 2021
│   │
│   ├── models/
│   │   ├── user_profile.py             ← Dataclass UserProfile + Diagnosis + Recommendation
│   │   └── rule.py                     ← Dataclass Rule + Condition
│   │
│   ├── utils/
│   │   ├── calculators.py              ← Funciones financieras puras (ratios, DAI, proyección)
│   │   ├── validators.py               ← Validación de rangos y tipos (llamada desde app.py)
│   │   └── logger.py                   ← Configuración de logging centralizada
│   │
│   └── requirements.txt
│
├── frontend/
│   ├── index.html                      ← SPA: wizard 4 pasos + dashboard, atributos data-i18n
│   └── assets/
│       ├── css/
│       │   ├── main.css                ← Variables CSS, dark mode ([data-theme="dark"]), controles
│       │   ├── wizard.css              ← Progreso, pasos y secciones del wizard
│       │   └── dashboard.css           ← Semáforo, métricas, recomendaciones, dark mode overrides
│       ├── js/
│       │   ├── theme.js               ← Toggle claro/oscuro, persistencia en localStorage
│       │   ├── i18n.js                ← Traducciones ES/EN (~80 claves), applyTranslations()
│       │   ├── api.js                 ← fetchDiagnosis(), fetchExplain(), checkHealth()
│       │   ├── wizard.js              ← Navegación, validación por paso, buildProfile()
│       │   ├── dashboard.js           ← renderDashboard(), métricas y situación con i18n
│       │   ├── charts.js              ← Chart.js: proyección 6 meses, colores adaptados al tema
│       │   └── explainer.js           ← Panel "¿Por qué?" por regla (llama a /explain)
│       └── img/
│           └── logo.svg               ← Logo vectorial minimalista (polyline + circle)
│
├── tests/
│   ├── conftest.py                     ← Fixtures: perfiles crítico, en_riesgo, saludable
│   ├── unit/
│   │   ├── test_rules_diagnostico.py
│   │   ├── test_rules_deuda.py
│   │   ├── test_rules_ahorro.py
│   │   ├── test_rules_perfil.py
│   │   ├── test_rules_inversion.py
│   │   ├── test_rules_pronostico.py
│   │   ├── test_working_memory.py
│   │   ├── test_agenda.py
│   │   └── test_conflict_resolver.py
│   └── integration/
│       ├── test_engine_scenarios.py    ← Escenarios end-to-end: crítico / en riesgo / saludable
│       ├── test_engine_components.py   ← WorkingMemory, ConflictResolver, Agenda, Explainer
│       └── test_api_endpoints.py       ← Pruebas de /diagnose, /explain y /health
│
├── architecture/
